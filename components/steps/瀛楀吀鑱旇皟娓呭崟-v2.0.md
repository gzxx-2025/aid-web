# 字典联调清单（v2.0）

适用范围：创作流程中所有“下拉选择 / 类型枚举 / 官方词库”字段。  
后端变更基线：旧接口 `/api/user/prompt/getData` 下线，改为：

- 官方只读参数词库
  - `POST /api/user/prompt/official/category/list`
  - `POST /api/user/prompt/official/item/list`
  - `POST /api/user/prompt/official/item/detail`
- 枚举字典
  - `POST /api/user/dict/enum/list`

---

## 1. 数据加载总入口（必须先通）

### 前端入口

- `composables/usePromptDictionary.ts`

### 实际加载流程

1. 调 `official/category/list` 获取分类白名单
2. 取 `categoryCode[]` 调 `official/item/list` 批量拉词条
3. 分别按 `enumTypes` 调 `dict/enum/list` 拉枚举并合并

### 联调验收点

- 登录态下调用均返回 `code=200`
- 分类列表非空（至少包含文档定义的官方分类）
- 词条列表字段完整（`id/categoryCode/itemName/promptText/promptTextEn/coverUrl/sortOrder`）
- 枚举返回结构为 `{ enumType, items: [{ value, desc }] }`

## 2. 页面点对点对照表

## A. 全局设置（第一步）

- 页面：`components/steps/GlobalSetting.vue`
- 使用字典：
  - 画幅比例：`aspectRatioEnumOptions` → `AspectRatioEnum`
  - 剧本类型：`scriptTypeEnumOptions` → `ScriptTypeEnum`
  - 创作方式：`creationModeEnumOptions` → `CreationModeEnum`（缺失时回退 `GenModeEnum`）
  - 分镜模式：`storyboardModeEnumOptions` → `StoryboardModeEnum`
  - 精选风格：`styleLibraryCards` → 官方词库 `categoryCode=style`（当前后端若无 style 分类则为空）
- 后端接口：
  - `POST /api/user/dict/enum/list`
  - `POST /api/user/prompt/official/category/list`
  - `POST /api/user/prompt/official/item/list`
- 前端过滤值（需后端值域一致）：
  - AspectRatio: `16:9 / 9:16 / 4:3 / 3:4 / 1:1 / 21:9`
  - ScriptType: `story / live-commentary`
  - CreationMode: `image-to-video / multi-parameter`
  - StoryboardMode: `single / nine-grid`

### 本页验收点

- 各下拉都能渲染中文 label，保存时 value 正确回填
- 后端返回超集时，前端只展示上述合法值
- `selectedStyle` 在字典更新后可通过名称自动对齐新 id

## B. 分镜生成参数面板（图像侧）

- 页面：`components/steps/StoryboardGeneratePanel.vue`
- 页面：`components/steps/StoryboardLeftPanel.vue`
- 使用词库分类：
  - `composition`（构图）
  - `shot_size`（景别）
  - `camera_angle`（角度）
  - `focal_length`（焦距）
  - `color_tone`（色调）
  - `lighting`（光线）
  - `exposure_blur`（摄影技法）
  - `camera_movement`（镜头运动）
  - `shooting_technique`（特殊拍摄手法）
- 后端接口：
  - `POST /api/user/prompt/official/category/list`
  - `POST /api/user/prompt/official/item/list`

### 本页验收点

- 每个下拉均出现“无 + 官方词条列表”
- 选中项能稳定保存 `{ key=id, value=itemName }`
- 历史旧 key（英文标识）可自动迁移到新 id（不丢用户配置）
- 排序按 `sortOrder` 升序

## C. 分镜生成弹窗（脚本/视频）

- 页面：`components/steps/StoryboardGenerateModal.vue`
- 使用枚举：
  - 比例：`AspectRatioEnum`（二次过滤仅保留 `16:9/9:16/4:3/1:1`）
  - 模型类型：`ModelTypeEnum`
  - 镜头密度：`ShotDensityEnum`
  - 分辨率：`VideoResolutionEnum`
  - 音效：`AudioSourceEnum`
- 后端接口：
  - `POST /api/user/dict/enum/list`

### 本页验收点

- 枚举项渲染后可选，提交参数值与后端枚举 value 一致
- 若某枚举类型后端暂未开放，不影响其余项加载（当前实现为分类型容错）
- 默认值（如 `720p`、`none`）必须存在于后端枚举，否则前端应可提示/降级

## 4. 风险与注意事项

- `style` 已从“官方只读参数词库”职责中剥离；若后端不再返回该分类，`GlobalSetting` 的精选风格将为空列表（这是预期行为，不是报错）。
- `ShotDensityEnum` 与 `VideoResolutionEnum` 不在后端当前文档白名单内，如后端未提供会导致对应选项为空。建议后端确认是否纳入 `enum/list` 白名单。
- `enumTypes` 大小写敏感，联调时必须使用准确枚举名。
- 仅允许官方只读词库，不应再出现个人词库混入。
