# 统一字典 `POST /api/user/prompt/getData` — 后端配置对齐说明

前端已取消「接口无数据时的本地静态选项」：对应分类或枚举未配置时，界面展示 **「暂无数据」** 或 **「加载中…」**。请按下表在后台维护数据。

## 一、提示词库 `promptLibList`（`promptType`）

| `promptType` | 说明 | 前端使用位置 |
|--------------|------|----------------|
| `style` | 视频风格 | 全局设置 · 精选风格库 |
| `composition` | 构图 | 分镜生成面板、左侧栏 |
| `shot_size` | 景别 | 同上 |
| `camera_angle` | 拍摄角度 | 同上 |
| `focal_length` | 焦距 | 同上 |
| `color_tone` | 色调 | 同上 |
| `lighting` | 光线 | 同上 |
| `exposure_blur` | 曝光与虚化（界面文案「摄影技法」） | 同上 |
| `camera_movement` | 镜头运动 | 分镜视频生成面板 |
| `shooting_technique` | 拍摄技巧（界面「特殊拍摄手法」） | 同上 |

单条需：`id`、`promptType`、`promptName`、`status`（`0` 正常）、可选 `coverUrl`、`sortOrder`。

## 二、枚举 `enumList`（`enumType` + `value` + `desc`）

| `enumType` | 建议 `value` 示例 | 说明 | 前端使用位置 |
|------------|-------------------|------|----------------|
| `AspectRatioEnum` | `16:9`、`9:16`、`4:3`、`3:4`、`1:1`、`21:9` | 全局比例；其中 `16:9/9:16/4:3/1:1` 会进入分镜视频弹窗比例下拉 | 全局设置、StoryboardGenerateModal |
| `ScriptTypeEnum` | 与作品表单约定一致（如 `story`、`live-commentary`） | 须与 `GlobalSettingData.scriptType` 白名单一致 | 全局设置 |
| `CreationModeEnum` | 如 `image-to-video`、`multi-parameter` | 无数据时会再读 `GenModeEnum` | 全局设置 |
| `GenModeEnum` | 与创作模式业务约定一致 | 仅当 `CreationModeEnum` 为空时作为备选 | 全局设置 |
| `StoryboardModeEnum` | 如 `single`、`nine-grid` | 须与白名单一致 | 全局设置 |
| `ModelTypeEnum` | 与持久化 `videoModel` 字符串一致 | 生视频模型卡片 | StoryboardGenerateModal |
| `ShotDensityEnum` | `concise`、`standard`、`detailed` | 镜头密度 | StoryboardGenerateModal（脚本） |
| `VideoResolutionEnum` | `720p`、`1080p` | 视频清晰度 | StoryboardGenerateModal（视频） |
| `AudioSourceEnum` | `none`、`with-sound` | 音效 | StoryboardGenerateModal（视频） |

`desc` 为界面展示文案；`value` 须与上表及前端类型约定一致，否则会被过滤掉而不展示。

## 三、非字典项（仍由前端写死）

- **全局设置 · 模型策略**（省钱优先 / 画质优先）：当前未对接 `getData`，与字典无关。

## 四、旧数据兼容（仅逻辑，不展示静态列表）

若历史保存的是英文 key（如 `rule_of_thirds`），前端会通过内置 **旧 key → 中文名** 映射，在接口返回的 `promptName` 中匹配并升级为数字 `id`，无需在后端保留英文 key。
